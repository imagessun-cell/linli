> 迭代类型：体验优化 / 功能新增 / 假数据支持 / 百度地图集成

## 1. 迭代背景

当前 V1.0 版本（MVP）中，用户首次进入小程序/H5 后，必须立即选择“服务者”“用工方”或“家属”身份才能使用核心功能。这一设计在前期验证阶段是合理的，便于快速划分角色并展示对应界面。但在实际测试和用户访谈中发现存在以下问题：

1. **认知负担高**：首次进入的用户（尤其是银发服务者和潜在用工方）并不清楚自己应选哪个身份，甚至可能因为选错而无法正常使用。
2. **缺乏吸引力**：用户看不到平台上的具体岗位，无法形成“这里有很多机会”的第一印象，导致注册转化率偏低。
3. **不符合常见就业平台习惯**：大多数招聘/零工平台（如美团众包、兼职猫、58同城）都是先展示岗位列表，用户感兴趣后再引导登录/完善身份。

因此，需要**弱化强制身份选择**，新增**基于地理位置的岗位首页**，让用户先浏览、后操作，提升首次访问的友好度和转化率。

## 2. 迭代目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 提升首页停留时长 | 首页平均停留时长 | 增加 30% |
| 提升岗位点击率 | 岗位卡片点击率（点击查看详情） | ≥ 15% |
| 提升注册转化率 | 从浏览岗位到完成注册/登录的比例 | 提升 20% |
| 降低身份选错率 | 身份选错后切换身份的比例 | 降低 50% |

## 3. 功能变更概览

### 3.1 新增首页（岗位广场）

- **页面路径**：`/`（原身份选择页不再作为首页，改为独立页面或引导弹窗）
- **核心内容**：
  - 基于用户地理位置（需授权）展示附近 5 公里内的**待接单任务**（岗位）
  - 岗位卡片展示：任务类型图标、标题、距离、报酬、体力等级、发布者评分
  - 支持按距离、报酬、体力等级排序/筛选
  - 支持按任务类型（陪诊、陪聊、保洁、做饭等）筛选
  - 顶部搜索框：按地址或关键词搜索岗位
- **未登录状态**：可浏览岗位列表，点击“立即接单”或“查看详情”时引导登录/注册
- **已登录状态**：根据用户已有身份（服务者/用工方）展示不同操作按钮
  - 如果身份为**服务者**：岗位卡片上直接显示“接单”按钮
  - 如果身份为**用工方**：显示“查看详情”按钮（用于了解市场行情，或切换身份）
  - 如果身份为**家属**：显示“代长辈接单”按钮（需先绑定长辈）

### 3.2 身份选择后置

- **移除**：首次启动强制选择身份
- **保留**：“我的”页面中的身份切换入口（服务者/用工方/家属），用户可以随时切换
- **新增**：在首页点击接单/发单操作时，若当前身份无权限（例如用工方点了接单），则弹出提示并引导切换身份或完善资料

### 3.3 地图视角（百度地图）

- **V1.1 版本实现**：在岗位列表基础上增加 **地图模式** 按钮，切换后可在地图上以标注形式展示附近岗位。
- 使用 **百度地图 JavaScript API v3.0** 和 **百度地图 WebGL 版**。
- 用户点击地图标注，弹出信息窗（InfoWindow）展示岗位概要，并可点击“查看详情”跳转。

### 3.4 岗位详情页优化

- 原有任务详情页进行适配：增加“分享岗位”“收藏岗位”功能（后续迭代）
- 增加“相似岗位推荐”模块

## 4. 详细功能设计

### 4.1 岗位卡片设计

每个岗位卡片包含以下字段：

| 字段 | 说明 | 样例 |
|------|------|------|
| 任务类型图标 | 根据 type 显示不同 Emoji 或 SVG | 🩺 陪诊 |
| 标题 | 简短描述 | “陪李爷爷去人民医院复查” |
| 距离 | 用户当前位置到任务地址的直线距离（调用百度地图测距 API 或前端计算） | 350m |
| 报酬 | 显示金额 | ¥45 |
| 体力等级 | 轻度/中度/重度，使用色块区分 | 🟢 轻度 |
| 发布者评分 | 用工方平均评分 | ⭐4.9 |
| 任务状态标签 | 仅显示“待接单” | （不显示已接单任务） |
| 操作按钮 | 根据登录状态和角色动态显示 | “立即接单” / “查看详情” |

### 4.2 排序与筛选

- **排序方式**：
  - 默认：距离最近（由近及远）
  - 可切换：报酬最高、体力等级（轻度优先）
- **筛选条件**：
  - 任务类型：多选（陪诊、陪聊、保洁、做饭、接送、看护、跑腿、助教等）
  - 体力等级：多选（轻度、中度）—— 不推荐重度
  - 报酬范围：滑块（预设 20-200 元）

### 4.3 地理位置获取与百度地图集成

#### 4.3.1 前端地理位置获取

- 使用原生浏览器 Geolocation API 获取用户经纬度（WGS-84 坐标）。
- 若用户拒绝授权，则回退到 IP 定位（调用百度地图 IP 定位 API 或第三方服务）。
- 若仍失败，展示默认城市（如“北京市”）。

#### 4.3.2 坐标转换

- 百度地图使用 BD-09 坐标系，而原生获取为 WGS-84。需通过**百度地图坐标转换 API** 进行转换（`BMap.Convertor`）或调用 `/api/location/convert` 后端代理。
- 前端可统一使用 `BMapGL.Convertor.translate(point, 3, callback)`（3 表示 WGS-84 转 BD-09）。

#### 4.3.3 距离计算

- **前端展示距离**：使用百度地图 API 提供的 `BMapGL.Map.getDistance(start, end)` 计算两点之间的直线距离（单位：米），并格式化为“公里/米”。
- **后端排序**：因百度地图坐标系统一为 BD-09，后端计算距离时需使用 Haversine 公式（适用于地球曲率）。可直接应用 BD-09 经纬度计算近似距离（误差在可接受范围内）。

#### 4.3.4 地图模式

- 在岗位广场页面增加“列表/地图”切换按钮。
- 地图模式：
  - 初始化百度地图，设置中心为用户当前位置（若未授权则定位到默认城市）。
  - 根据后端返回的附近岗位列表，在地图上添加标注（Marker）。
  - 点击标注弹出 InfoWindow，显示岗位类型、报酬、距离，并提供“查看详情”链接。
  - 地图支持缩放、拖拽，实时刷新当前视野内的岗位（可选，V1.1 可先不实现动态加载）。

## 5. 技术实现方案（百度地图版）

### 5.1 后端变更

#### 5.1.1 新增接口

| 方法 | 路径 | 说明 | 请求参数 | 响应 |
|------|------|------|----------|------|
| GET | `/api/tasks/nearby` | 获取附近岗位列表（分页），使用百度坐标（BD-09）计算距离 | `latitude`, `longitude`, `radius`, `page`, `pageSize`, `sortBy`, `filters` | 任务列表，每个任务附加 `distance` 字段 |
| GET | `/api/tasks/:taskId` | 岗位详情（公开） | `taskId` | 任务详情，包含发布者信息 |
| GET | `/api/location/convert` | 坐标转换代理（WGS-84 → BD-09） | `lat`, `lng`, `from=1(表示WGS-84), to=5(表示BD-09)` | `{ lat, lng }` 转换后坐标 |

#### 5.1.2 数据库支持

- 确保 `t_task` 表的 `latitude` 和 `longitude` 存储的是 **BD-09 坐标系** 的经纬度。
- 后端距离计算函数（Node.js 示例）：
  ```javascript
  function getDistance(lat1, lon1, lat2, lon2) {
    const R = 6371; // km
    const dLat = (lat2 - lat1) * Math.PI / 180;
    const dLon = (lon2 - lon1) * Math.PI / 180;
    const a = Math.sin(dLat/2) * Math.sin(dLat/2) +
              Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) *
              Math.sin(dLon/2) * Math.sin(dLon/2);
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    return R * c * 1000; // 返回米
  }
  ```

新增 `/api/mock/init` 接口（仅开发环境），调用假数据初始化脚本，方便重置演示数据。

```javascript
// 获取用户位置（WGS-84）
function getUserLocation() {
  return new Promise((resolve, reject) => {
    navigator.geolocation.getCurrentPosition(
      pos => resolve({ lat: pos.coords.latitude, lng: pos.coords.longitude }),
      err => reject(err)
    );
  });
}

// 转换为百度坐标（调用后端代理或前端API）
async function convertToBD09(lat, lng) {
  // 方法1：后端代理
  const res = await axios.get('/api/location/convert', { params: { lat, lng } });
  return res.data;
  // 方法2：前端调用百度API（需AK）
  // 详见百度地图坐标转换文档
}
```

#### 5.2.3 地图模式实现

- **初始化地图**：
  ```javascript
  const map = new BMapGL.Map('mapContainer');
  const point = new BMapGL.Point(lng, lat);
  map.centerAndZoom(point, 15);
  map.enableScrollWheelZoom(true);
  ```
- **添加标注**：遍历岗位列表，为每个岗位创建标注，并绑定信息窗。
- **距离计算**：可使用 `map.getDistance(startPoint, endPoint)` 获取前端显示距离，也可用后端返回的 `distance`。

#### 5.2.4 首页默认视图

- 用户进入首页后，首先尝试获取地理位置并显示列表模式。
- 点击“地图”按钮时，切换到地图模式，并调用地图 API 展示标注。

### 5.3 部署注意事项

- **百度地图 AK**：需要在百度地图开放平台申请浏览器端密钥（AK），并配置在 `.env` 前端构建变量中。
- **坐标转换 QPS**：免费版每日调用限制，建议使用后端代理并缓存转换结果。
- **假数据**：仅用于开发环境，生产环境需移除假数据接口。

| 点击“地图”按钮，地图显示标注且位置正确 | 标注分布在对应坐标上 |
| 点击标注弹窗，包含岗位概要信息 | 能正确跳转详情页 |

| 后端坐标转换接口 | 输入 WGS-84，返回 BD-09 坐标，偏差在 50m 内 |

## 11. 附录：百度地图 AK 申请与配置

1. 访问 [百度地图开放平台](http://lbsyun.baidu.com/) 注册并登录。
2. 进入“控制台” → “应用管理” → “创建应用”。
3. 应用类型选择“浏览器端”，白名单填写 `localhost` 和实际部署域名（如 `linli.com`）。
4. 获取 AK 后，在前端项目环境变量文件中添加：
   ```
   VITE_BAIDU_MAP_AK=您的AK
   ```
5. 在代码中使用 `import.meta.env.VITE_BAIDU_MAP_AK` 引用。
