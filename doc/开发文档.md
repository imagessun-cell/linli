# 邻里 H5 开发文档（本地部署版）

> 版本：V1.0  
> 更新日期：2026年5月29日  
> 适用对象：前端、后端、测试开发人员  
> **技术栈调整**：原小程序版改为 H5（Vue3） + Node.js（Express） + SQLite，仅本地部署，无需上线。  

---

## 1. 项目技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端（用户端 H5）** | Vue 3 + Element Plus + Axios + Vite | 响应式设计，支持 PC 和移动端浏览器 |
| **前端（管理后台）** | Vue 3 + Element Plus + Axios + Vite | 独立项目，为社区管理员和平台运营提供后台 |
| **后端** | Node.js 18.x + Express 5.x | 提供 RESTful API，支持跨域，JWT 鉴权 |
| **数据库** | SQLite 3.43.0 | 嵌入式数据库，单文件存储，无需独立服务 |
| **文件存储** | 本地文件系统（开发环境） | 头像、语音、课程视频等存储在 `./uploads` 目录 |
| **即时通讯** | WebSocket（Socket.IO） | 实时消息推送（订单状态更新、聊天） |
| **第三方服务** | 微信支付（可选，本地可 Mock）<br>腾讯位置服务（API）<br>微信公众平台（可选） | 本地部署时可忽略或 Mock |

## 4. 数据库设计（SQLite）

### 4.1 数据库初始化脚本

创建 `database/init.sql`：

```sql
-- 启用外键约束
PRAGMA foreign_keys = ON;

-- 用户表
CREATE TABLE IF NOT EXISTS t_user (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    openid TEXT UNIQUE,                     -- 微信openid（可选）
    phone TEXT NOT NULL UNIQUE,
    nickname TEXT,
    avatar_url TEXT,
    role INTEGER NOT NULL DEFAULT 1,        -- 1-服务者 2-用工方 3-家属
    status INTEGER DEFAULT 0,               -- 0-正常 1-冻结
    real_name TEXT,
    id_card TEXT,                           -- 加密存储
    face_verified INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- 服务者扩展表
CREATE TABLE IF NOT EXISTS t_worker (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL UNIQUE,
    age INTEGER NOT NULL,                   -- 限制50-65
    community TEXT NOT NULL,
    service_radius INTEGER DEFAULT 3000,    -- 米
    skills TEXT NOT NULL,                   -- JSON数组
    emergency_contact_name TEXT,
    emergency_contact_phone TEXT,
    total_orders INTEGER DEFAULT 0,
    total_hours INTEGER DEFAULT 0,
    avg_rating REAL DEFAULT 5.0,
    honor_level INTEGER DEFAULT 0,          -- 0-3
    status INTEGER DEFAULT 0,               -- 0-审核中 1-已认证 2-驳回
    FOREIGN KEY (user_id) REFERENCES t_user(id)
);

-- 用工方表
CREATE TABLE IF NOT EXISTS t_employer (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL UNIQUE,
    credit_score INTEGER DEFAULT 100,
    common_addresses TEXT,                  -- JSON
    FOREIGN KEY (user_id) REFERENCES t_user(id)
);

-- 家属绑定关系表
CREATE TABLE IF NOT EXISTS t_agent_binding (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_user_id INTEGER NOT NULL,         -- 家属用户ID
    worker_user_id INTEGER NOT NULL,        -- 被绑定的服务者用户ID
    verified INTEGER DEFAULT 0,             -- 0-待确认 1-已确认
    created_at TEXT NOT NULL,
    FOREIGN KEY (agent_user_id) REFERENCES t_user(id),
    FOREIGN KEY (worker_user_id) REFERENCES t_user(id)
);

-- 任务表
CREATE TABLE IF NOT EXISTS t_task (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employer_id INTEGER NOT NULL,
    type INTEGER NOT NULL,                  -- 1陪诊 2陪聊 3小时保洁 4做饭 5接送 6看护 7跑腿 8助教 9其他
    start_time TEXT NOT NULL,
    end_time TEXT NOT NULL,
    duration_minutes INTEGER NOT NULL,
    address TEXT NOT NULL,
    latitude REAL NOT NULL,
    longitude REAL NOT NULL,
    physical_level INTEGER DEFAULT 1,       -- 1轻度 2中度 3重度
    budget REAL NOT NULL,
    is_charity INTEGER DEFAULT 0,
    special_requirements TEXT,
    status INTEGER DEFAULT 0,               -- 0待接单 1已接单 2服务中 3已完成 4已取消 5争议中
    worker_id INTEGER,
    created_at TEXT NOT NULL,
    expires_at TEXT NOT NULL,
    FOREIGN KEY (employer_id) REFERENCES t_user(id),
    FOREIGN KEY (worker_id) REFERENCES t_user(id)
);

-- 订单表
CREATE TABLE IF NOT EXISTS t_order (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER NOT NULL UNIQUE,
    order_no TEXT NOT NULL UNIQUE,
    employer_id INTEGER NOT NULL,
    worker_id INTEGER NOT NULL,
    total_amount REAL NOT NULL,
    platform_commission REAL DEFAULT 0,
    worker_income REAL NOT NULL,
    points_earned INTEGER DEFAULT 0,
    status INTEGER DEFAULT 1,               -- 1待服务 2服务中 3待确认 4已完成 5已取消 6退款中
    insurance_no TEXT,
    start_service_time TEXT,
    finish_service_time TEXT,
    created_at TEXT NOT NULL,
    pay_time TEXT,
    complete_time TEXT,
    FOREIGN KEY (task_id) REFERENCES t_task(id),
    FOREIGN KEY (employer_id) REFERENCES t_user(id),
    FOREIGN KEY (worker_id) REFERENCES t_user(id)
);

-- 钱包表
CREATE TABLE IF NOT EXISTS t_wallet (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    worker_id INTEGER NOT NULL UNIQUE,
    cash_balance REAL DEFAULT 0,
    points_balance INTEGER DEFAULT 0,
    frozen_amount REAL DEFAULT 0,
    FOREIGN KEY (worker_id) REFERENCES t_worker(id)
);

-- 钱包流水表
CREATE TABLE IF NOT EXISTS t_wallet_transaction (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    wallet_id INTEGER NOT NULL,
    type INTEGER NOT NULL,                  -- 1收入 2提现 3兑换
    amount REAL,
    points_change INTEGER,
    order_id INTEGER,
    status INTEGER DEFAULT 0,               -- 0处理中 1成功 2失败
    created_at TEXT NOT NULL,
    FOREIGN KEY (wallet_id) REFERENCES t_wallet(id)
);

-- 课程表
CREATE TABLE IF NOT EXISTS t_course (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    video_url TEXT NOT NULL,
    duration_seconds INTEGER,
    cover_url TEXT,
    sort_order INTEGER DEFAULT 0,
    is_required INTEGER DEFAULT 0
);

-- 学习记录表
CREATE TABLE IF NOT EXISTS t_learning_record (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    worker_id INTEGER NOT NULL,
    course_id INTEGER NOT NULL,
    progress_percent INTEGER DEFAULT 0,
    finished INTEGER DEFAULT 0,
    last_watch_time TEXT,
    FOREIGN KEY (worker_id) REFERENCES t_worker(id),
    FOREIGN KEY (course_id) REFERENCES t_course(id)
);

-- 银发圈动态表
CREATE TABLE IF NOT EXISTS t_community_post (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    worker_id INTEGER NOT NULL,
    content_type INTEGER DEFAULT 1,         -- 1语音 2图文
    content_text TEXT,
    voice_url TEXT,
    image_urls TEXT,
    like_count INTEGER DEFAULT 0,
    comment_count INTEGER DEFAULT 0,
    status INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    FOREIGN KEY (worker_id) REFERENCES t_worker(id)
);

-- 索引
CREATE INDEX idx_user_phone ON t_user(phone);
CREATE INDEX idx_user_role ON t_user(role);
CREATE INDEX idx_task_status_created ON t_task(status, created_at);
CREATE INDEX idx_task_employer ON t_task(employer_id);
CREATE INDEX idx_order_worker ON t_order(worker_id);
CREATE INDEX idx_order_employer ON t_order(employer_id);
CREATE INDEX idx_wallet_worker ON t_wallet(worker_id);
```

### 4.2 数据库连接配置（SQLite）

使用 `better-sqlite3` 或 `sqlite3` 驱动，在 `db/index.js` 中初始化：

```javascript
const sqlite3 = require('sqlite3').verbose();
const path = require('path');
const fs = require('fs');

const dbPath = path.resolve(__dirname, '../data/linli.db');
const db = new sqlite3.Database(dbPath);

// 执行初始化脚本（首次运行）
const initSQL = fs.readFileSync(path.resolve(__dirname, 'init.sql'), 'utf8');
db.exec(initSQL, (err) => {
  if (err) console.error('数据库初始化失败:', err);
  else console.log('数据库初始化成功');
});

module.exports = db;
```


错误码：0成功，400参数错误，401未授权，403无权限，404资源不存在，500服务器错误。

### 5.5 中间件示例

**JWT 验证中间件**（`middleware/auth.js`）：

```javascript
const jwt = require('jsonwebtoken');
module.exports = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ code: 401, message: '未登录' });
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (err) {
    return res.status(401).json({ code: 401, message: 'token无效' });
  }
};
```

### 6.3 路由守卫

根据用户角色（`user.role`）和是否实名认证，动态跳转。

```javascript
router.beforeEach((to, from, next) => {
  const user = useUserStore();
  if (to.path !== '/login' && !user.token) {
    next('/login');
  } else if (user.role === 'worker' && !user.verified && to.path !== '/worker/apply') {
    next('/worker/apply');
  } else {
    next();
  }
});
```

## 7. 本地部署步骤

### 7.1 环境要求

- Node.js 18.x 或更高
- npm 或 yarn
- Git（可选）

### 7.2 后端启动

1. 克隆项目或创建文件夹 `backend`。
2. 进入 `backend`，运行 `npm init -y`，安装依赖：
   ```bash
   npm install express cors dotenv jsonwebtoken bcryptjs sqlite3 multer socket.io
   npm install -D nodemon
   ```
3. 创建 `.env` 文件，配置 `PORT=3000`、`JWT_SECRET`。
4. 创建 `db` 目录和 `init.sql`、`index.js` 如上。
5. 在 `app.js` 中编写基础 Express 服务：
   ```javascript
   const express = require('express');
   const cors = require('cors');
   const dotenv = require('dotenv');
   dotenv.config();
   const app = express();
   app.use(cors());
   app.use(express.json());
   app.use('/uploads', express.static('uploads'));
   // 挂载路由...
   const server = app.listen(process.env.PORT, () => console.log(`Server running on port ${process.env.PORT}`));
   // Socket.IO 初始化
   const io = require('socket.io')(server, { cors: { origin: '*' } });
   ```
6. 运行 `nodemon app.js`（开发环境）或 `node app.js`。
7. 访问 `http://localhost:3000` 测试 API。

### 7.3 前端启动

1. 进入 `frontend` 目录，运行：
   ```bash
   npm create vite@latest . -- --template vue
   npm install element-plus axios pinia vue-router @vitejs/plugin-legacy
   ```
2. 配置 `vite.config.js` 代理跨域：
   ```javascript
   export default {
     server: {
       proxy: {
         '/api': 'http://localhost:3000',
   ```

## 8. 关键业务流程实现说明

### 8.1 发单与接单

- 用工方发布任务 → 后端写入 `t_task`，状态 `0`。
- 服务者获取推荐任务：基于 `physical_level`（推荐轻度为主）、距离（SQLite 使用 Haversine 公式计算经纬度距离）、技能匹配。
- 接单：后端检查任务状态和距离，更新 `task.worker_id` 和状态为 `1`，同时创建 `t_order` 记录，冻结资金（Mock 中直接扣减用工方预支付虚拟余额）。

### 8.2 服务流程与保险

- 服务者点击“开始服务”：写入 `order.start_service_time`，调用 Mock 保险接口生成 `insurance_no`。
- 完成服务：上传验收图片，状态变为待确认。
- 用工方确认：将订单状态改为已完成，调用结算逻辑：增加服务者钱包余额，增加积分。

### 8.3 消息通知

- 使用 Socket.IO 广播订单状态变更给相关用户。
- 即时聊天：每个聊天室可存储消息到 SQLite（可选）。

## 10. 常见问题

**Q：SQLite 数据库文件位置在哪？**  
A：默认在 `backend/data/linli.db`，首次启动会自动创建。

**Q：如何清除所有数据重新开始？**  
A：删除 `linli.db` 文件，重启后端，会自动执行 `init.sql` 重建表。

**Q：如何 Mock 微信支付？**  
A：在支付接口直接返回成功，不实际调用微信接口。冻结资金使用虚拟账户模拟。

**Q：如何实现地理位置计算距离？**  
A：SQLite 不支持空间函数，可以在后端使用 JavaScript 计算两个经纬度之间的直线距离（Haversine 公式），再排序。
